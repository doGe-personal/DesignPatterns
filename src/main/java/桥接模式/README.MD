# 桥接模式
## 定义
    桥接模式是将抽象部分与它的实现部分分离，
    使它们都可以独立地变化。它是一种对象结构型模式，
    又称为柄体(Handle and Body)模式或接口(Interfce)模式。。
## 优缺点
    - 优点
        将实现予以解耦，让它与界面之间不再永久绑定
        抽象和实现独立拓展，不会影响对方
        对于"具体的抽象类"所做的改变，不会影响到客户
    - 缺点
        增加复杂度
    - 用途
        适合使用在需要跨越多个平台的图形和窗口系统上
        当需要不同的方式改变接口和实现时。
